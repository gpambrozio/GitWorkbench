01

Overview & principles

The Workbench shows the working-tree state of the active repository and lets the user stage, review, and commit changes without leaving the helper app. It mirrors the mental model of Tower / Fork / GitHub Desktop, rendered with native macOS materials.

Principles

  • Native first. System grays, vibrancy/translucency on popovers, SF font, standard traffic-light chrome. It should feel like it shipped with the OS, not like a web view.
  • Everything visible, nothing modal. Branches, file list, and diff are always on screen. Switching files or branches never opens a new window.
  • Staging is the spine. The staged ⁄ unstaged split is the primary axis. Commit is always one glance away in the same column as the files it acts on.
  • Read before you write. The diff pane is the largest region — reviewing changes is the most common task, committing is the rare one.
Scope note. This is a plain git view — no assistant-authored-change labelling or AI commit messages in this spec. That can layer on later without disturbing the structure described here.

03

Anatomy of the panes

Toolbar

Unified titlebar (no separate title row). Left cluster carries the traffic lights + repository name against a 1px divider. The center-left group holds the remote actions and branch switcher; the trailing group holds the diff-mode segmented control.

  • Pull / Push show pending counts inline (e.g. Pull 1, Push 2) when behind/ahead.
  • Fetch is a no-count refresh of remote refs.
  • Branch pill shows a branch glyph + current branch name + chevron; opens the branch menu.
  • History and Stash open their respective workspace views (rail selection mirrors this).
  • Segmented control toggles Unified ↔ Split diff; persists per-repo.

Pane 1 — Rail

Grouped source list on the deepest gray. Three sections: Workspace (Changes · History · Stashes, each with a count), Branches (local branches; the current one is bold with a HEAD tag), and Remotes (expandable to remote-tracking branches). Single-click selects; a branch row offers Switch / Merge / Rename / Delete on right-click.

Pane 2 — Changes list + commit

Two collapsible groups — Staged then Changes — each with a count badge and a bulk action (Stage all / Unstage all). Section headers are sticky while scrolling. Each row shows: a stage checkbox, the status glyph, filename, dimmed directory, and a +adds / −dels chip that swaps to a Discard affordance on hover. The commit composer is docked to the bottom of this pane so it never scrolls away.

Pane 3 — Diff

A header strip (status glyph · filename · path · status word · Stage/Unstage · Discard) above the scrolling hunk view. Empty when no file is selected.

04

File status model

Every changed file carries one status. The glyph is a rounded square: outlined in the list when unselected, filled when the row is selected, in the status color.

MModifiedtracked, edited
AAddednew, staged
DDeletedremoved
RRenamedmoved / renamed
UUntrackednew, unstaged

Line stats render in monospace with tabular figures: additions in green +24, deletions in red −6. A file may appear in both Staged and Changes groups when it has partially-staged hunks (future: hunk-level staging).

06

Diff viewer

Unified

Single column. Two gutter columns (old line ⁄ new line), a sign column (+ − ·), then code. Added lines tint green, removed tint red, each with a 3px colored edge bar. Default for narrow widths.

Split

Side-by-side old vs new with an aligned center divider. Changed cells tint; empty counterpart cells get a faint hatched fill. Best on wide windows for line-level comparison.

  • Hunk headers (@@ −14,8 +14,9 @@) separate regions with a faint accent-tinted band.
  • Code is set in the system monospace at 12 / 20 px; gutters use the tertiary ink and are non-selectable.
  • Deleted files render their content as a fully-removed block; binary/image files fall back to a before→after metadata row (future).
  • Large files lazy-render hunks; "Load more" appears past a threshold rather than blocking on the full patch.

States & edge cases

StateTreatment
Clean treeChanges list shows a centered "Working tree clean" empty state; diff pane echoes it; commit disabled.
No file selectedDiff pane shows file glyph + "Select a file to view changes".
Nothing stagedCommit button reads Commit to {branch}, disabled and gray.
Detached HEADBranch pill shows short SHA + a "detached" tag; commit warns it won't be on a branch.
Merge conflictConflicted files get a ! glyph (orange-red) and sort to the top; diff shows conflict markers with "Use ours / Use theirs" actions.
Sync in flightToolbar remote buttons disable; spinner toast; result toast on completion.
Push rejectedError toast (red) suggesting Pull first; no data lost.
Very long pathsDirectory truncates with a middle ellipsis; filename never truncates before the directory.
